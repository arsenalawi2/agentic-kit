# Architecture Page Template

Create an `/architecture` route that visualizes the tech stack of every project the user works on — an "Enterprise Architecture Repository" as a single-page app.

## Reference Implementation

A complete, working Vue 3 implementation is available as a reference. **Use this as your starting template** — copy it into the project and adapt the data source. Do not try to rebuild from scratch.

Full single-file component:
- `templates/reference/ArchitecturePage.vue` — complete file (2,856 lines)

Split into parts (read only what you need):
- `templates/reference/architecture-template.html` — HTML template (530 lines)
- `templates/reference/architecture-script.js` — Vue 3 script + vis-network + d3 (1,379 lines)
- `templates/reference/architecture-styles.css` — all CSS including responsive + dark mode (945 lines)

For a full build, copy `ArchitecturePage.vue` directly. For adapting or debugging a specific part, read the split files.

## Data Source

**Primary (auto-updated):** Fetch from `/tech-stack.json` in the project's public directory. This file is auto-generated by the leaderboard hook after every Claude Code session — it scans `package.json`, `requirements.txt`, `docker-compose.yml`, etc. Hand-curated fields (purpose, status, ports, cross-cutting infra) are preserved across re-scans.

```javascript
const res = await fetch('/tech-stack.json')
const data = await res.json()
```

## Page Structure

Use the EZ Design System. The page is a long scroll with alternating section backgrounds.

### 1. Hero Section
- Badge: "Enterprise Architecture Repository"
- Large title: "Project Architecture"
- Subtitle: "Every project. Every technology. Every connection."
- Stats row: total projects, unique technologies, languages, currently running
- CTA link: "Explore the Graph" scrolling to network section

### 2. Inventory — Project Cards
- Section label: "Inventory"
- Title: "Every project, every stack"
- Category tabs at top for filtering (e.g., "Dashboard", "Bot", "Data Pipeline")
- Expand all / Collapse all buttons
- Grid of collapsible project cards, each showing:
  - **Header (always visible):** chevron, project name, status badge (running/stopped/partial), category, tech count, port count, path
  - **Body (when expanded):** purpose description, port pills, stack groups by category
  - **Tech detail inline panel:** when a tech item is clicked, expand a detail panel inside the card showing: name, role, description, vendor, license, maturity pill, cost, lock-in risk pill, adoption status, specs grid, alternatives tags, depends-on tags, used-by projects tags, used-for tags, docs link
- Status badges color-coded: running = green, stopped = default, partial = warning

### 3. Shared Technology Grid
- Section label: "Shared Technology"
- Title: "What spans the most projects"
- Alternating background (section-alt)
- Grid of tech cards sorted by project count (most shared first)
- Each card: colored dot (category color), tech name, project count, list of project chips
- Clicking a tech card scrolls up and opens its detail in the relevant project card

### 4. Network Graph (vis-network)
- Section label: "The Map"
- Title: "Architecture network"
- Interactive network graph using vis-network library
- Project nodes = large circles, Technology nodes = small circles
- Edges connect projects to their technologies
- Category filter buttons at top (each with colored dot)
- Click a node to focus it — everything unrelated fades to gray
- Reset View button
- Legend: square = project, circle = technology
- Selected node detail panel below graph: project shows purpose + full tech list, technology shows which projects use it

### 5. Sankey Flow Chart (d3)
- Section label: "Cross-Project Flow"
- Title: "Projects -> technologies"
- Alternating background (section-alt)
- d3-based flow visualization: projects on left, top N technologies on right, curved connections between them
- Hover/click to highlight a single node's connections
- Shows dependency density visually

### 6. Stats Footer — By the Numbers
- Section label: "By the Numbers"
- Title: "Architecture at a glance"
- Numbers grid: total projects, unique technologies, programming languages, database engines, AI providers, currently running
- Language breakdown: horizontal bar chart with language name, bar, count

### 7. How This Was Built (Meta)
- Section label: "Meta"
- Title: "How this page was built"
- Alternating background (section-alt)
- Brief description of how it was built with Claude Code
- Vibe stats row: session count, prompts, projects scanned, lines of code, model used, lines hand-written
- Pipeline steps: Scan → Curate → Render (numbered steps with descriptions)
- "Built with" tags row

### 8. Footer
- Generated timestamp
- How to refresh data

## Auto-Detection Script

Include a `scan_projects.py` script that walks project directories and detects:
- **Languages:** from package.json (JS/TS), requirements.txt/pyproject.toml (Python), Cargo.toml (Rust), go.mod (Go), *.swift (Swift)
- **Frontend frameworks:** from dependencies (vue, react, svelte, angular)
- **Backend frameworks:** from dependencies (fastapi, flask, django, express)
- **Databases:** from docker-compose.yml services, dependencies (pg, sqlite3, mongoose)
- **AI providers:** from dependencies (anthropic, openai, google-generativeai)
- **Infrastructure:** from docker-compose.yml, Tailscale configs
- **Build tools:** from devDependencies (vite, webpack, esbuild)

The script outputs JSON that the user reviews and enriches with hand-curated metadata (purpose, status, ports, cross-cutting infra).

## Key Interactions

1. **Project card expand/collapse** — click header to toggle, with smooth transition
2. **Tech detail inline** — click a tech item within a project card to expand its detail panel *inside that card* (not at page bottom)
3. **Network graph click-to-focus** — clicking a node grays out unrelated nodes/edges
4. **Category filter** — toggle category buttons to show/hide tech in the network graph
5. **Shared tech → project card** — clicking a shared tech card scrolls to and opens the relevant detail

## Design Notes

- Use the EZ Design System throughout (tokens, components, utilities)
- Section backgrounds alternate: bg-primary / section-alt (bg-secondary)
- Section pattern: uppercase label (10px, accent color) + title (24px) + description + content
- Stats use Space Grotesk display font, large sizes (32-48px)
- Status pills: running = green bg, stopped = gray bg, partial = amber bg
- Maturity pills: Stable = green, Emerging = blue, Legacy = gray
- Lock-in risk pills: Low = green, Medium = amber, High = red
- Port pills: mono font, small, outlined
- Tech category colors should be consistent across sections (graph, shared grid, project cards)
- Dark mode: toggle via html.dark class, all tokens switch
- Responsive: stack to single column on mobile, hide network graph details on small screens

## Dependencies
